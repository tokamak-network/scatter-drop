## 트랙 C — 후속 기능 (post-v1, 설계상 단계적)

DESIGN §10 빌드오더의 3~6단계. v1 범위 밖, 수요 따라 우선순위.

| # | 기능 | 무게 | 비고 |
|---|------|------|------|
| C1 | **ONCHAIN_SNAPSHOT 자동화** | 중 | 지금은 Dune 수동. 인덱서(The Graph/RPC) 붙여 "토큰+블록+조건"→명단 자동 |
| C2 | **GatedDrop** (온체인 실시간 검증) | 대 | 별도 컨트랙트. claim 시 보유/스테이킹 직접 검증. DESIGN §5.2(B) |
| C3 | **소셜·태스크 자격** | 대 | 백엔드+OAuth+sybil. 트위터/디스코드 퀘스트 |
| C4 | **배포 방식 확장** | 중 | 베스팅(cliff+linear), 선착순(FCFS). 디자인 types.ts에 모델 있음 |
| C5 | **가스리스 claim** | 중 | Permit2/relayer. 소액 에어드랍 claim 가스 부담 해소 |
| C6 | **멀티체인** | 대 | L2 배포, 체인별 proof/레지스트리 |
| C7 | 알림 (이메일/지갑) | 중 | 클레임 가능 알림. 지갑↔이메일 매핑(프라이버시) |
| C8 | 콘솔 경고 정리 (pino-pretty/ox externals) | 소 | next.config webpack externals. 무해하나 깔끔 |
